THE SUMMONING

git clone https://github.com/aday1/The-DAW-Horsemen-of-the-apocalypse-MCP-survival-Pack.git DAW-Horsemen cd DAW-Horsemen CARE.bat :: pull-if-behind + heal OSC/paths/Mackie + shortcut + health

CARE always heals. INSTALL is deps+heal; UPDATE is git pull+heal. Wire IDEs with IDE_SETUP.txt. X-Touch: packages/mackie-xtouch/SETUP_MACKIE.txt (MCU MIDI beside MCP — not a second MCP server).

THE PROPHECY (UPDATES)

UPDATE.bat :: checks GitHub, pulls if behind, refreshes deps

The repo is the source of truth. Dev happens in the clone; when a version is worthy, it is committed and PUBLISH.bat-ed. Every other machine just runs UPDATE.bat. Loose copies are heresy.

ONE WAR, MANY GENERALS

Bitwig's OSC controller speaks to one process. The pack ships a shared HTTP-SSE server — run_bitwig_mcp_shared.bat — so Claude CLI, Cursor, VS Code MCP extensions and Claude Desktop all command the same battlefield at http://127.0.0.1:8080/sse instead of fighting over UDP 9001. Full IDE wiring: IDE_SETUP.txt.
