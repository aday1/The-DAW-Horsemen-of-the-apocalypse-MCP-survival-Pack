MACKIE / BEHRINGER X-TOUCH (with DAW Horsemen MCP)
==================================================
DEFAULT OWNER: BITWIG (DrivenByMoss MCU — best supported in this pack)

The X-Touch is NOT a separate MCP server. It talks MCU/MIDI to ONE DAW.
MCP (Cursor/Claude) talks OSC/HTTP to every DAW. No MIDI fight there.

  X-Touch  --MCU MIDI-->  Bitwig (MCU - Control Universal)   <-- default
  Agents   --MCP------->  Bitwig + REAPER + Renoise

DawpocalypseMCP.bwextension (DrivenByMoss fork) includes:
  - MCU - Control Universal  (+ extenders)   <-- use this for full X-Touch
  - X-Touch One
  - HUI variants
  - DawpocalypseMCP OSC (for agents)         <-- keep ON always


CRITICAL -- WINDOWS MIDI IS EXCLUSIVE
-------------------------------------
Only ONE app can open the X-Touch MIDI In/Out ports.

Bitwig error "not enough memory" / "Could not open MIDI" for X-Touch =
REAPER (or another app) already holds the ports. Not RAM.

CARE/heal DISABLES REAPER's Mackie MCU surface so Bitwig can own the desk.
Backup: %APPDATA%\REAPER\reaper.ini.bak_horsemen_xtouch
Note:    %APPDATA%\REAPER\horsemen_xtouch_owner.txt


BITWIG SETUP (default)
----------------------
1. CARE.bat (or Launchers F) — heals extension + frees X-Touch from REAPER
2. Settings -> Controllers -> Add Controller
     Vendor: DrivenByMoss / DawpocalypseMCP build
     Product: MCU - Control Universal
3. MIDI In/Out = X-Touch (hardware ports)
4. Keep DawpocalypseMCP OSC on: Receive 8005 / Send 9001 / 127.0.0.1
5. If MIDI error after heal: Controllers -> MCU OFF/ON, or unplug/replug X-Touch
6. Template (heal installs):
     Documents\Bitwig Studio\Library\Templates\Mackie-XTouch-Behringer-Yggdrasil


REAPER (agents only by default — no X-Touch)
--------------------------------------------
REAPER MCP bridge still works (file mode). Mackie surface is OFF so it
does not steal X-Touch from Bitwig.

To put X-Touch on REAPER instead (optional):
  1. Bitwig: disable MCU / X-Touch controller (leave OSC on)
  2. Restore from reaper.ini.bak_horsemen_xtouch OR
     Preferences -> Control/OSC/Web -> Add -> Mackie Control Universal
  3. Know that Bitwig MCU will fail until REAPER surface is removed again


RENOISE
-------
No Mackie MCU in this pack. Drive Renoise via ReMCP; keep X-Touch on Bitwig.


HEAL / CARE
-----------
  Always: Bitwig = X-Touch owner, REAPER Mackie cleared, templates installed.
