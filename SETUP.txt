THE DAW HORSEMEN MCP PACK -- TLDR SETUP
========================================

You get four DAW paths in one repo:

  REAPER    -> packages/reaper-mcp/
  RENOISE   -> packages/renoise-mcp-bridge/   (+ Renoise ReMCP running inside Renoise)
  BITWIG    -> packages/bitwig-mcp-server/   (+ DrivenByMoss / OSC in Bitwig)
  REASON    -> packages/reason/              (no separate MCP: use REAPER + Reason Rack Plugin)

Per DAW (do only what you use):

1) REAPER
   - Copy packages/reaper-mcp/reaper_mcp_bridge.lua into %APPDATA%\REAPER\Scripts\
   - REAPER: Actions -> Load ReaScript -> Run that file
   - pip install -r packages/reaper-mcp/requirements.txt
   - Cursor MCP: python packages/reaper-mcp/reaper_mcp_server.py (see package README for env)

2) RENOISE
   - Install/start the Renoise ReMCP tool so HTTP MCP is up (often port 19714)
   - cd packages/renoise-mcp-bridge && npm install
   - Cursor MCP: node bridge.js with cwd = that folder

3) BITWIG
   - Install DrivenByMoss (or your OSC build), add OSC controller, Receive 8005 / Send 9001 match env
   - cd packages/bitwig-mcp-server && uv sync   (or pip install -e .)
   - Cursor MCP: python -m bitwig_mcp_server with cwd = that folder

4) REASON
   - Open Reason inside REAPER as Reason Rack Plugin on a track; control the session with REAPER MCP.

Optional: PowerShell script scripts/extract_to_machine.ps1 copies Bitwig server into your Bitwig Extensions folder.

Dashboard env (optional): REAPER_MCP_DASHBOARD=1, RENOISE_MCP_DASHBOARD=1, BITWIG_MCP_DASHBOARD=1

Install help tools (no DAW needed): get_reaper_mcp_install_guide, get_bitwig_mcp_install_guide

SHARED BITWIG SERVER (v2)
-------------------------
Run packages\bitwig-mcp-server\run_bitwig_mcp_shared.bat (ONE instance).
Every MCP client then connects to http://127.0.0.1:8080/sse -- no more
port 9001 fights between Cursor / Claude CLI / Desktop.
Details: packages/bitwig-mcp-server/SHARED_SERVER.md

INSTALL / UPDATE
----------------
Fresh machine: git clone ... DAW-Horsemen, then INSTALL.bat
Updates:       UPDATE.bat   (git pull + deps refresh)
Dev machine:   edit, test, commit, PUBLISH.bat
